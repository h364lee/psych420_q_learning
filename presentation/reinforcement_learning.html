Understanding Reinforcement Learning

  1. Autonomous agents learn through trial and error to make decisions by interacting with their environment.
  1. Utilize positive and negative feedback to maximize cumulative rewards.
  1. Q-Learning

Key Terms

Entities

  • Agent
  • Environment

Functions

  • Actions (A)
  • State (S)
  • Reward (R)
  • Policy (off-policy)

 

Need to learn without labeled data — In many real-world problems, there is no correct answer provided. The agent must learn from rewards or penalties instead of explicit supervision. Need to handle sequential decision-making — Actions influence future states and future rewards. Decisions cannot be evaluated in isolation. Need to operate in unknown or partially known environments — The agent does not have access to full transition dynamics. Learning must occur through interaction with the environment. Need to balance exploration vs exploitation — The agent must try new actions (exploration) while using known good actions (exploitation). This trade-off is central to effective learning.

Provides a framework for learning through interaction — Closely models how humans and animals learn from consequences. Connects with principles from psychology and neuroscience. Enables autonomous decision-making systems — Systems can act, learn, and improve without explicit programming. Applicable to robotics, recommendation systems, and control systems. Introduces foundational concepts in AI — Value functions (how good a state/action is), Policies (decision rules), Reward signals (objective of learning), Trial-and-error learning.

Tutorial: Maze Setup

import numpy as np
import matplotlib.pyplot as plt

# 0 = free cell
# 1 = wall
# 2 = hole (terminal, negative reward)
# 3 = goal (terminal, positive reward)

maze = np.array([
    [0, 0, 1, 0, 0, 0],
    [0, 2, 0, 0, 1, 0],
    [0, 0, 0, 2, 0, 0],
    [0, 1, 1, 0, 1, 0],
    [0, 0, 0, 0, 0, 0],
    [2, 0, 1, 0, 0, 3]
])

ROWS, COLS = maze.shape
START = (0, 0)
GOAL  = (5, 5)

Tutorial: Initialize Q-Table & Parameters

# shape: (rows, cols, 4 actions)
Q = np.zeros((ROWS, COLS, 4))

# action directions: 0=up, 1=down, 2=left, 3=right
DIRS = [(-1,0), (1,0), (0,-1), (0,1)]

alpha   = 0.5   # learning rate
gamma   = 0.9   # discount factor
epsilon = 0.2   # exploration rate
episodes = 500

Tutorial: Step Function


def step(state, action):
    r, c = state
    dr, dc = DIRS[action]
    nr, nc = r + dr, c + dc

    # if move goes out of bounds or into wall, stay still
    if nr < 0 or nr >= ROWS or nc < 0 or nc >= COLS or maze[nr, nc] == 1:
        nr, nc = r, c

    # assign reward based on what the next cell is
    tile = maze[nr, nc]
    if tile == 3:
        reward = 10       # reached goal
    elif tile == 2:
        reward = -1       # fell in a hole
    else:
        reward = -0.04    # small penalty to encourage efficiency

    done = (tile == 3 or tile == 2)
    return (nr, nc), reward, done

Tutorial: Choose Action Function

def choose_action(state):
    r, c = state
    if np.random.random() < epsilon:
        return np.random.randint(4)           # random action
    else:
        return np.argmax(Q[r, c])             # best known action

Tutorial: Training Loop

steps_per_episode = []
deltas_over_time  = []

for episode in range(episodes):
    state = START
    steps = 0

    while True:
        r, c = state

        action = choose_action(state)
        next_state, reward, done = step(state, action)
        nr, nc = next_state

        # compute TD error
        delta = reward + gamma * np.max(Q[nr, nc]) - Q[r, c, action]

        # update Q table
        Q[r, c, action] += alpha * delta

        deltas_over_time.append(delta)
        steps += 1
        state = next_state

        if done or steps > 500:
            break

    steps_per_episode.append(steps)

Libraries: Gymnasium

Python library for developing and comparing RL algorithms

  • Pre-built environments
  • Same interface for every environment: reset(), step(), render()
  • Same algorithm across environments without modification
import gymnasium as gym

env = gym.make("LunarLander-v2")
state, info = env.reset()

action = env.action_space.sample()
state, reward, terminated, truncated, info = env.step(action)

Libraries: Stable Baselines3 - DQN

Neural network instead of Q-table

  • Same TD error algorithm but Q-table replaced by a multilayer perceptron
  • Two key additions: experience replay and target network
from stable_baselines3 import DQN
import gymnasium as gym

env   = gym.make("CartPole-v1")
model = DQN("MlpPolicy", env, verbose=1, gamma=0.99)
model.learn(total_timesteps=100_000)

Libraries: NeuroGym - RL on Cognitive Tasks

RL framework in cognitive tasks

  • Example: Go/No-Go task — a cognitive neuroscience task
  • Agent sees a stimulus and must decide whether to respond or withhold
  • fixation → stimulus → decision → reward
  • Trained agent behaviour can be compared directly to neural recordings
